# Установка прослойки подписки Remnawave

Прослойка встаёт между клиентами и панелью: отдаёт страницу подписки и конфиги, умеет блокировки/оверрайды, грейс, лог, вебхуки, чат, приманку и админку. Два сценария установки.

---

## Сценарий 1. Отдельный сервер (прослойка одна)

Прослойка живёт на своём домене и сама держит TLS.

```bash
git clone https://github.com/Mrvibecodic/remnawave-subscription-middleware.git
cd remnawave-subscription-middleware
sudo bash install.sh
```
На первом вопросе выберите **`1` — отдельный сервер**. Скрипт ставит nginx + php-fpm + certbot, выпускает сертификат и поднимает всё на `443`. В конце — ссылка на мастер `https://<домен>/admin/`.

---

## Сценарий 2. Рядом с панелью (основной)

Условие: панель Remnawave и её reverse-proxy (напр. **eGames**) уже стоят, домен подписки сейчас обслуживает контейнер `remnawave-subscription-page`. Прослойка встанет **перед** этим контейнером: добавит свои фичи на клиентском пути, а страницу подписки в браузере будет отдавать **через него** (по умолчанию режим «внешний контейнер»). Контейнер останавливать не нужно, вендоренный бандл не используется.

Скрипт ставит **только свои зависимости** на хост и **не трогает докер/конфиги панели** — последний шаг (правку nginx панели) вы делаете руками по этой инструкции.

### Шаг 1. Поставить прослойку
```bash
git clone https://github.com/Mrvibecodic/remnawave-subscription-middleware.git
cd remnawave-subscription-middleware
sudo bash install.sh
```
На первом вопросе выберите **`2` — рядом с панелью**. Скрипт спросит:
- **домен подписки** (тот же, что в `SUB_PUBLIC_DOMAIN` панели);
- **URL панели** — по умолчанию `http://127.0.0.1:3000` (панель опубликована на хосте; внутренний адрес, мимо Cloudflare);
- **локальный порт** прослойки — по умолчанию `8080`;
- **тип БД** — по умолчанию SQLite.

Он поставит nginx + php-fpm, поднимет прослойку на `http://127.0.0.1:8080` и запишет в настройки: режим «панель», URL панели и режим страницы «внешний контейнер» (`http://127.0.0.1:3010`). TLS и свой 443 он **не трогает**.

### Шаг 2. Направить домен подписки на прослойку (правка nginx ПАНЕЛИ)
Нужно, чтобы трафик домена подписки уходил на прослойку (`127.0.0.1:8080`) вместо контейнера subpage (`127.0.0.1:3010`).

**eGames (nginx):** в конфиге nginx панели (обычно `/opt/remnawave/nginx.conf`) поменяйте апстрим подписки — одна строка:
```nginx
upstream json {
    server 127.0.0.1:8080;
}
```
(было `server 127.0.0.1:3010;`). Сам server-блок домена подписки и его заголовки трогать не нужно — `X-Forwarded-Proto`, `X-Real-IP`, `X-Forwarded-For` там уже выставлены правильно.

Перезагрузите nginx панели:
```bash
docker exec <nginx-контейнер-панели> nginx -t && docker exec <nginx-контейнер-панели> nginx -s reload
```

**Общий случай (не eGames):** найдите в конфиге nginx панели `server`-блок вашего домена подписки (он проксирует на subscription-page, порт `3010`) и поменяйте его `proxy_pass`/upstream на `http://127.0.0.1:8080`. Заголовки `X-Forwarded-*` оставьте как есть.

> Контейнер `remnawave-subscription-page` **оставляем работать** — прослойка проксирует на него страницу подписки (режим «внешний контейнер»). Если subpage у вас на другом адресе/порту, поправьте его в админке → «Подключение». Бандл при этом не нужен.

### Шаг 3. Завершить мастер
Откройте `https://<домен-подписки>/admin/`. Режим «основная подписка (панель)» и URL панели уже подставлены — задайте логин/пароль админки и API-токен панели (Settings → API Tokens), и установка завершена.

### Проверка
```bash
# конфиг клиенту (UA приложения) — ждём 200:
curl -A "Happ/1.0" -o /dev/null -w "%{http_code}\n" https://<домен-подписки>/<shortUuid>
```
В браузере та же ссылка должна показать страницу подписки из панели.

---

## Заметки
- **`SUB_PUBLIC_DOMAIN`** в `.env` панели уже равен домену подписки (eGames его выставил) — менять не нужно.
- Прослойка ходит к панели по **`http://127.0.0.1:3000`** (локально, без Cloudflare) — так задумано; это внутренний трафик хоста.
- В eGames sub-vhost стоит `proxy_intercept_errors on; ... @redirect (444)` — на штатных ответах прослойки (200) это не мешает; трогать не обязательно.
